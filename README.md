# for people — Telegram Mini App

Личная книга контактов с AI-поиском по интересам, психологическим портретом и т.д.
Этот проект — готовый к деплою React-сайт, который открывается внутри Telegram
как Mini App.

Если ты не программируешь — этот файл можно целиком отдать фрилансеру
как техническое задание. Всё нужное уже внутри проекта.

---

## Что уже сделано

- Дизайн и вся логика перенесены из прототипа в обычный React-проект (Vite)
- Хранение данных переключено с внутреннего механизма Claude на
  `Telegram.WebApp.CloudStorage` (см. `src/storage.js`), с fallback на
  `localStorage`, если открыть сайт в обычном браузере не через Telegram
- Вызов AI-помощника переписан так, чтобы API-ключ никогда не попадал
  в код фронтенда — вместо этого используется backend-прокси
  (пример готов в `api/ai-proxy.js`)

## Что нужно сделать разработчику (или тебе, по шагам ниже)

### 1. Установить зависимости и проверить, что всё работает локально

```
npm install
npm run dev
```

Откроется адрес вроде `http://localhost:5173` — в обычном браузере
приложение тоже откроется (просто будет использовать `localStorage`
вместо Telegram CloudStorage, это нормально для проверки).

### 2. Настроить AI-помощника (необязательно, но без этого шага кнопка
    "AI помощник" будет показывать сообщение "ещё не подключён")

AI не может стучаться в Anthropic API напрямую с телефона пользователя —
это небезопасно (ключ был бы виден всем). Поэтому нужен маленький backend.

Самый простой вариант — Vercel:

1. Получи API-ключ на console.anthropic.com
2. Задеплой проект на Vercel (см. шаг 3) — папка `api/` подхватится
   автоматически как serverless-функция
3. В настройках проекта на Vercel: **Settings → Environment Variables**
   → добавь `ANTHROPIC_API_KEY` = твой ключ
4. Скопируй `.env.example` в `.env`, впиши туда адрес:
   `VITE_AI_PROXY_URL=https://твой-проект.vercel.app/api/ai-proxy`
5. Пересобери проект (`npm run build`) и передеплой

Если AI-помощник не нужен вообще — можно просто не настраивать этот шаг,
остальное приложение будет работать полностью (контакты, категории,
теги, психологический портрет, импорт/экспорт CSV, массовое редактирование).

### 3. Задеплоить сайт (получить https-ссылку)

Проще всего — Vercel:

1. Зарегистрируйся на vercel.com (можно через GitHub)
2. Залей этот проект к себе на GitHub
3. На Vercel: **New Project** → выбери репозиторий → Deploy
4. После деплоя получишь ссылку вида `https://for-people-xxxx.vercel.app`

Обязательно должно быть **https** — Telegram не откроет обычный http-сайт.

### 4. Создать бота и подключить Mini App

1. Напиши в Telegram боту **@BotFather**
2. `/newbot` → придумай имя и юзернейм → получишь токен (сохрани его)
3. `/mybots` → выбери своего бота → **Bot Settings** → **Menu Button**
   → **Configure Menu Button** → вставь ссылку с шага 3
   (или через `/newapp`, если хочешь полноценный Mini App с иконкой
   в общем каталоге приложений Telegram)

### 5. Проверить

Открой бота в Telegram → нажми кнопку меню внизу экрана — приложение
откроется прямо внутри Telegram, с адаптацией под тему (тёмная/светлая)
и нативными жестами.

## Важные технические ограничения (стоит знать)

- **Хранилище (CloudStorage)**: у Telegram есть лимит — 4096 символов на
  один ключ и максимум 1024 ключа на пользователя. Наш `storage.js` уже
  разбивает большие данные на кусочки автоматически, так что для обычного
  использования (десятки-сотни контактов) хватит с запасом. Если база
  вырастет до тысяч контактов с фото — стоит перейти на настоящую БД
  (например, Supabase — бесплатный тариф вполне подойдёт).
- **Фото профиля**: хранятся как есть внутри контакта (base64), тоже
  через чанкованное CloudStorage — работает, но если фото много и они
  крупные, может потребоваться Supabase/Firebase Storage вместо
  CloudStorage для самих картинок.
- **AI-помощник**: требует backend-прокси (см. шаг 2). Без него всё
  остальное приложение работает нормально.

## Структура проекта

```
for-people-miniapp/
├── index.html          — подключает Telegram Web App SDK
├── src/
│   ├── main.jsx         — точка входа, инициализация Telegram.WebApp
│   ├── App.jsx           — всё приложение (компонент, дизайн, логика)
│   └── storage.js        — обёртка над Telegram CloudStorage / localStorage
├── api/
│   └── ai-proxy.js       — пример backend-прокси для AI (Vercel Function)
├── .env.example          — куда вписать адрес AI-прокси
└── package.json
```
